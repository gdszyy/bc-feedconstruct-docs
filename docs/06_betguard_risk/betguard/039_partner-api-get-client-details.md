## 文档正文
GetClientDetails

The **GetClientDetails** method is called by FeedConstruct when the Partner calls either [CreateBet](../documentation?currentLoc=betGuard&location=createBet) or [GetMaxBetAmount](../documentation?currentLoc=betGuard&location=get_max_bet_amount).

It uses the [AuthToken](../documentation?currentLoc=betGuard&location=authToken) generated by the Partner and returns the player`s data **if the token is valid and the player is logged in** to the Partner’s website.

If the player is **not logged in**, the method must return **ErrorCode 1008**. For other issues—such as an **expired or invalid token**—relevant error codes should be returned, as specified in the [Error Codes](../documentation?currentLoc=betGuard&location=errors_returned_by_the_partner) section.

**Important:**The client`s currency must match one of the supported values listed in the [Currency Codes](../documentation?currentLoc=betGuard&location=currency_codes) section.

**Important:**The client`s currency should **remain unchanged throughout the client’s lifecycle**.

URL: /GetClientDetails

Example: **http://partnerhostname:port/api/GetClientDetails**

Where **http://partnerhostname:port/api** is the Partner’s Base URL

Method: POST

- Request parameters

| Field Name Field Name | Type Type | Requirement Requirement | Description Description |  |
| --- | --- | --- | --- | --- |
| AuthToken | String | Mandatory | [See AuthToken](/documentation?currentLoc=betGuard&location=authToken)  for details |
| TS | Long | Mandatory | [See TS parameter](/documentation?currentLoc=betGuard&location=ts_parameter_timestamp)  for details |
| Hash | String | Mandatory | [See Hash parameter](/documentation?currentLoc=betGuard&location=partner_api_hash_parameter)  for details |

- Response parameters

| Field Name Field Name | Type Type | Requirement Requirement | Description Description |  |
| --- | --- | --- | --- | --- |
| Login | String | Mandatory, max length - 255 | Client’s Login on the Partner’s side. The value of the ExternalID can be used instead |
| CurrencyId | String | Mandatory, max length - 3 | User’s currency code. |
| ExternalId | String | Mandatory, max length - 255 | Unique ID of the Client in the Partner’s Backend |
| LanguageId | String | Optional, max length - 2 | Two letter code of the Client’s language. For example “en” – for English |
| Email | String | Optional, max length - 255 | Email address of the Client. |
| BirthDate | Date | Optional | Birth date of the Client. |
| Gender | Int | Optional | 1 – for Male, 2 – for Female. |
| CurrentIp | String | Optional, max length - 50 | IP address of the client currently logged in to the Partner’s website. |
| Phone | String | Optional, max length - 255 | Primary phone number of the Client |
| ErrorCode | String | Optional | [See Error Codes](/documentation?currentLoc=betGuard&location=errors_returned_by_the_partner)  for valid values. Mandatory in case of error. |
| ErrorText | String | Optional | Error description |
| CountryId | String | Optional, max length - 2 | ISO 3166-1 alpha-2 code of the registration country |
| PartnerFlag | String | Optional, max length - 50 | Optional flag field to differentiate some clients from others |
| SportsProfileID | Int | Optional | Identifier for the user’s assigned sportsbook profile. |
| RMTblocked | Bool | Optional | Indicates if Risk Management Tool (RMT) is blocked for this user (true = blocked, false = allowed). |

**Note:** Optional fields can be omitted from the response. However, the fields **Gender, BirthDate, CurrentIp, Phone**, and **Email** are used for advanced player risk management (e.g., detecting multi-accounting or linked accounts, assigning additional categories, etc.) in the **Dedicated** package and are recommended to be included in the response if available.
