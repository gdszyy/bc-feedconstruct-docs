## 文档正文
Rollback

The **Rollback** method is triggered when a bet needs to be canceled due to one of the following reasons:

- The **Partner responds with an error** to the [BetPlaced](../documentation?currentLoc=betGuard&location=partner_api_bet_placed) call.
- The **Partner does not respond within 5 seconds** to the [BetPlaced](../documentation?currentLoc=betGuard&location=partner_api_bet_placed) call.

This call is sent by **FeedConstruct** to inform the Partner that the bet should be **canceled and not processed further**.

Handling Duplicate or Repeated Calls

- If the Partner has **not processed** the original [BetPlaced](../documentation?currentLoc=betGuard&location=partner_api_bet_placed) call with the given **TransactionId**, this **Rollback** can be **safely ignored**.
- If the Partner has **already canceled** the transaction, this call can also be **safely ignored**.

**Important:** In both cases, the Partner must still return **a valid success response** to prevent FeedConstruct from retrying the call.

URL: /Rollback

Example: **http://partnerhostname:port/api/Rollback**

Where **http://partnerhostname:port/api** is the Base URL for all calls.

Method: POST

| ExternalId | Long | Optional | The Partner’s ID of the bet. Is sent if received in the initial CreateBet call |
